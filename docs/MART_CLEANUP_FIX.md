# mart_toronto Table Cleanup Fix

**Date**: 2026-03-04
**Status**: ✅ Complete and verified
**Commit**: `7877bca`

## Problem Statement

The `mart_toronto` schema contained **10 tables** instead of the intended **6 tables** due to incomplete cleanup of consolidated tables during previous sprints:

| Table | Status | Consolidation |
|-------|--------|----------------|
| `mart_neighbourhood_foundation` | ❌ Deprecated | Removed Sprint 13 (was subset of demographics) |
| `mart_neighbourhood_demographics` | ❌ Deprecated | Consolidated into `mart_neighbourhood_people` (Sprint 13) |
| `mart_neighbourhood_amenities` | ❌ Deprecated | Consolidated into `mart_neighbourhood_people` (Sprint 13) |
| `mart_toronto_rentals` | ❌ Deprecated | Consolidated into housing tables (removed 2026-02-23) |
| `mart_neighbourhood_housing_rentals` | ❌ Deprecated | Consolidated into `mart_neighbourhood_housing` (Sprint 13) |

These tables still existed in the database but were no longer generated by dbt.

## Root Cause

- **Consolidation logic**: Intermediate and mart models were refactored to combine tables
- **No cleanup mechanism**: dbt `run` only creates/updates defined models; it doesn't drop old tables
- **Manual cleanup missed**: Previous sprints removed model files but forgot to clean the database

## Solution

### 1. Fixed Column Reference (dbt Model)

**File**: `dbt/models/marts/toronto/mart_neighbourhood_overview.sql`

**Change**:
```sql
-- Before
housing as (
    select
        neighbourhood_id,
        year,
        affordability_index  -- ❌ Column no longer exists
    from {{ ref('mart_neighbourhood_housing') }}
),
...
        h.affordability_index as affordability_score,

-- After
housing as (
    select
        neighbourhood_id,
        year,
        housing_affordability_index  -- ✅ Correct column name
    from {{ ref('mart_neighbourhood_housing') }}
),
...
        h.housing_affordability_index as affordability_score,
```

**Reason**: During the housing consolidation sprint, the column was renamed to be more descriptive. The overview model was not updated to use the new name.

### 2. Added Automatic Cleanup (ETL Script)

**File**: `scripts/data/load_toronto_data.py`

**Changes**:
1. **Added cleanup method** (`_cleanup_deprecated_tables`):
   - Drops 5 deprecated tables after dbt completes
   - Uses `IF EXISTS` to handle already-deleted tables
   - Runs in a separate transaction for each table
   - Logs each drop for audit trail
   - Catches exceptions gracefully

2. **Integrated into pipeline**:
   - Called after `dbt test` completes
   - Runs only in normal mode (skipped in `--dry-run`)
   - Uses existing session management

**Code**:
```python
def _cleanup_deprecated_tables(self) -> None:
    """Drop deprecated mart tables from previous consolidations."""
    from sqlalchemy import text

    deprecated_tables = [
        "mart_neighbourhood_foundation",
        "mart_neighbourhood_demographics",
        "mart_neighbourhood_amenities",
        "mart_toronto_rentals",
        "mart_neighbourhood_housing_rentals",
    ]

    try:
        with get_session() as session:
            for table_name in deprecated_tables:
                try:
                    session.execute(
                        text(f"DROP TABLE IF EXISTS mart_toronto.{table_name} CASCADE")
                    )
                    session.commit()
                    logger.info(f"  Dropped deprecated table: {table_name}")
                except Exception as e:
                    logger.warning(f"  Could not drop {table_name}: {e}")
                    session.rollback()
    except Exception as e:
        logger.warning(f"  Cleanup encountered error: {e}")
```

## Future Prevention

To prevent similar issues in the future:

1. **When consolidating tables**:
   - Update all downstream model references
   - Add explicit DROP statements in a follow-up PR (like this one)
   - Document the consolidation in CHANGELOG.md

2. **dbt best practices**:
   - Run `dbt lineage <model>` before making changes
   - Test with `dbt run` + `dbt test` after changes
   - Use `dbt docs` to validate contract changes

3. **Code review**:
   - Review `dbt lineage` output in PRs that touch staging/intermediate models
   - Flag any models that reference deleted columns
   - Require explicit cleanup statements for deprecated tables
